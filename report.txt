  1. PROJECT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CodeGuard AI is a full-stack web application that provides AI-powered
code reviews. Users paste code in any of 15+ programming languages,
and the AI (Google Gemini 3 Flash Preview) analyzes it for:

  • Bugs & Logic Errors
  • Security Vulnerabilities (OWASP Top 10)
  • Performance Bottlenecks
  • Architecture & Design Flaws

The AI acts as a "Lead Software Architect and Security Engineer" and
returns structured Markdown with:
  ## 🚩 Issues Found
  ## 🛡️ Security Risks
  ## ✅ Optimized Solution
  ## 💡 Architecture Tips

Users can then chat with the AI for follow-up questions, all within
a ChatGPT-style interface.

  TRIGGER: on_auth_user_created
    AFTER INSERT ON auth.users
    → INSERT INTO public.users (id, username, email)

  RLS POLICIES:
    users:   SELECT/UPDATE/INSERT where auth.uid() = id
    reviews: SELECT/INSERT/DELETE where auth.uid() = user_id
    chats:   SELECT/INSERT via review ownership

  8. CREDIT SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  MAX_CREDITS = 20 per user per day

  Logic:
    • Each user has: daily_count + last_review_date in users table
    • On review request:
      - If ghostMode = true → skip credit check, don't save to DB
      - If ghostMode = false:
          * Check if last_review_date = today
          * If same day: check daily_count < 20
          * If different day: reset count to 0
    • After successful review: increment daily_count, update date
    • Credits are per-user (isolated by user_id in database)

  Ghost Mode:
    • When ON: unlimited reviews, no DB save, no credit deduction
    • When OFF: normal credit system applies

  9. SECURITY MEASURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Supabase Row Level Security (RLS) on all tables
  2. JWT token verification on every API request
  3. User can only access their own reviews/chats
  4. Service role key only on backend (never exposed to frontend)
  5. CORS restricted to frontend URL only
  6. Input validation on all endpoints
  7. Code length limit (50,000 characters)
  8. Ghost Mode for privacy-conscious users
